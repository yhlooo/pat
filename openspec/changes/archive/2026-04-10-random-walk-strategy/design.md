## Context

交易系统已有完整的策略执行框架：`Strategy` 接口定义了 `Execute(ctx, status) ([]Action, meta, error)` 方法，`UpdownSeriesTrader` 每秒调用一次策略并处理返回的 Action。现有猴子策略（`Monkey`）用于测试，现在需要实现基于数学模型的实际交易策略。

当前 `trading.Market` 结构体缺少市场结束时间，策略无法判断市场何时关闭。`polymarket.Market` 已有 `EndDate` 字段，但未传递到策略可访问的 `Status` 中。

## Goals / Non-Goals

**Goals:**
- 实现基于随机游走数学模型的交易策略，利用正态近似计算涨跌概率
- 通过概率与市场价格的偏差发现套利机会
- 将市场结束时间暴露给策略层

**Non-Goals:**
- 不实现精确的二项分布计算（正态近似足够且性能更好）
- 不实现几何随机游走模型（仅使用算术随机游走）
- 不优化波动率估算方法（按文档要求使用简单的单次观测估算）
- 不支持非对称概率模型
- 不修改策略接口或交易执行框架

## Decisions

### 1. 市场结束时间传递方式

**选择**: 在 `trading.Market` 结构体中新增 `EndDate time.Time` 字段。

**理由**: 最直接的方式，策略通过 `status.CurrentMarket.EndDate` 即可获取。`polymarket.Market` 已有此数据，只需在构建 `trading.Market` 时传递即可。`rotateMarket` 中也需要设置新市场的 `EndDate`。

### 2. 概率计算方法

**选择**: 使用正态近似 `P(Yes) = 1 - Φ((K - S₀) / (σ√n))`，其中 Φ 通过 `math.Erf` 实现。

**理由**: 对于 n ≥ 30（5分钟市场有 300 秒，15分钟有 900 秒），正态近似精度足够。Go 标准库不提供正态 CDF，但 `Φ(x) = 0.5 * (1 + erf(x/√2))` 是标准转换。

**备选方案**: 使用二项分布精确计算。对于 n=300，计算量大且无必要。正态近似在此场景下误差可忽略。

### 3. 波动率估算

**选择**: σ = |当前价格 - 上次价格| / 距上次执行的秒数。在策略结构体中记录上次执行时间和价格。

**理由**: 遵循策略文档的明确要求。虽然单次观测噪声很大，但策略依赖的是概率偏差而非精确概率值，容忍一定噪声。

### 4. 交易决策逻辑

**选择**: 当 `P(Yes) - Price(Yes)` 超过 ±0.2 阈值时触发交易，优先卖出已有持仓。

**理由**:
- 买入 Yes 等价于卖出 No（都增加 Yes 头寸），优先卖出可利用已有持仓
- FOK 订单确保要么立即成交要么取消，不挂单
- 滑点保护设为 ±0.2 确保能成交

### 5. 策略结构体设计

```
RandomWalk struct {
    lastTradeTime time.Time       // 上次交易时间（控制 30s 间隔）
    lastExecTime  time.Time       // 上次执行时间（估算 σ）
    lastPrice     decimal.Decimal // 上次执行时的资产价格（估算 σ）
}
```

**理由**: 最小化状态。策略无需配置参数（阈值、金额等），按文档固定值实现即可。

## Risks / Trade-offs

- **[σ 估算不稳定]** 单次价差估算波动率非常粗糙 → 对于 5/15 分钟市场，偏差阈值 0.2 足够大，能容忍估算噪声
- **[正态近似误差]** n 较小时近似精度下降 → 市场结束前 30s 不交易，保证 n ≥ 30
- **[σ = 0]** 价格无变化时无法估算 → 跳过本轮交易，仅记录数据
- **[首次执行]** 无历史数据 → 跳过交易，仅记录当前时间和价格作为基准
