## Context

当前 `RandomWalk` 策略通过 `lastPrice` 和 `lastExecTime` 两个字段记录上一次执行时的价格和时间，波动率 σ 的估算公式为 `|S₀ - lastPrice| / Δt`，仅使用一个样本点。这种方式噪声极大，容易因瞬时波动产生错误的交易信号。

## Goals / Non-Goals

**Goals:**
- 使用近 1 分钟价格序列估算波动率，提高统计可靠性
- 自动清理过期数据，防止长时间运行时的内存泄漏
- 保持与现有 Strategy 接口的兼容性

**Non-Goals:**
- 不改变交易决策逻辑（阈值、滑点保护等）
- 不改变概率计算公式
- 不引入新的外部依赖

## Decisions

### 1. 数据结构：使用 slice 存储价格观测点

引入 `pricePoint` 结构体（`time time.Time`, `price decimal.Decimal`），使用 `[]pricePoint` slice 存储。

**替代方案：** ring buffer 或 `container/ring`。

**选择 slice 的理由：** Execute 大约每几秒调用一次，1 分钟内最多 20-30 个点，数据量极小。Slice 在此规模下性能足够，且代码可读性最好。清理过期数据时利用 slice 切割即可。

### 2. 波动率计算公式：Realized Variance

使用布朗运动模型下的 MLE 估计量：

```
σ = √( (1/m) * Σ( (Sᵢ - Sᵢ₋₁)² / (tᵢ - tᵢ₋₁) ) )
```

其中 m 为增量数量，求和遍历缓存中所有连续观测点对。

**替代方案：** 对数收益率标准差。

**选择理由：** 当前模型基于绝对价格差（非对数收益），保持一致性。该公式在 BM 假设下是 σ² 的最大似然估计，统计性质最优。

### 3. 最少数据点要求：2 个

缓存中至少需要 2 个观测点（即 1 个增量）才能估算 σ。少于 2 个点时不交易。

### 4. 缓存清理策略：每次 Execute 时清理

在每次 Execute 调用的开始阶段，删除时间戳早于 `now - 1 minute` 的观测点。通过遍历找到分界点，使用 slice 切割保留新部分。

**理由：** 保证过期数据及时清理，实现简单，开销可忽略（数据量极小）。

## Risks / Trade-offs

- **[短期内数据不足]** → 策略启动后前 1 分钟内数据点较少，波动率估计可能不稳定。这是可接受的——宁可少交易也不基于噪声信号交易。
- **[slice 偶尔的分配]** → 清理过期数据时 slice 切割可能产生新的底层数组。由于数据量极小（< 30 个元素），GC 压力可忽略。
